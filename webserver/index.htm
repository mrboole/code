Project 1: Submission Instruction, Grading Criteria and Self-testing Instruction

Please find the details of this project from this web page.

Note that you do not have to use Java. Any other language would be equally fine, provided that it has a socket interface which you use for this assignment. Java is recommended here because you may not easily find similar functionalities as in Java, such as multi-threading. And you're already provided with the source code for doing most part of the homework. It is important that you should NOT use any wrapper classes (like HTTP class already provided by some packages).


Submission Instruction

Submit the code via email to the TA. For ease of grading, please include instructions in the README file on platform used, any special compile, running instructions, etc. Please do not send multiple emails. The code must be well-documented. The grader will run your code on one host and test with a standard web browser from another host on the same network. The grader will use windows and Linux-based platforms for testing. In case of problems or if you used a very uncommon platform, you must be able to demonstrate the working of the server code on a mobile computer that you will bring.

Please submit your homework in a single gzipped tar file or a zip file (containing all your source code and README file) before the deadline. A filename "MyWebServer.java" is recommended if you use java as your programming language. No late submission will be admitted.

Self-testing Instruction

Here are some instructions to help you test your web server before submission.

1. Save the sgs.html file of this web page, the fiu.jpeg file of the bottom image and this PDF file (HwAndVirus.pdf) to your own computer. Change file name of sgs.html to index.html. Make sure you put these three saved files (index.html, fiu.jpeg, HwAndVirus.pdf) and your Web server program file IN THE SAME FOLDER.

 

2. Compile and run your Web server.

 

3. Visit http://localhost:6789/index.html via web browser on the same computer. If you use other port number for your Web server, you could simply change 6789 to the number you are using. NOTE: If you omit ":6789", the browser will assume port 80 which most likely will not have a server listening on it.

 

4. Try to access different files or paths to see if you can meet all the grading criterias listed above.
